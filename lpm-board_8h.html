void LpmEnterLowPower (void)
 This API shall be used by the application when there is no more code to execute so that the system may enter low-power mode. The mode selected depends on the information received from LpmOffModeSelection( ) and LpmSysclockRequest( ) This function shall be called in critical section.

void LpmEnterStopMode (void)
 This API is called by the low power manager in a critical section (PRIMASK bit set) to allow the application to implement dedicated code before entering Stop Mode.
 
void LpmExitStopMode (void)
 This API is called by the low power manager in a critical section (PRIMASK bit set) to allow the application to implement dedicated code before getting out from Stop Mode. This is where the application should reconfigure the clock tree when needed.
 
void LpmEnterOffMode (void)
 This API is called by the low power manager in a critical section (PRIMASK bit set) to allow the application to implement dedicated code before entering Off mode. This is where the application could save data in the retention memory as the RAM memory content will be lost.
 
void LpmExitOffMode (void)
 This API is called by the low power manager in a critical section (PRIMASK bit set) to allow the application to implement dedicated code before getting out from Off mode. This can only happen when the Off mode is finally not entered. In that case, the application may reverse some configurations done before entering Off mode. When Off mode is successful, the system is reset when getting out from this low-power mode.

This API notifies the low power manager if the specified user allows the Off mode or not. When the application does not require the system clock, it enters Stop Mode if at least one user disallow Off Mode. Otherwise, it enters Off Mode. The default mode selection for all users is Off mode enabled.
